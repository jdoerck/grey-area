# Glyph PNG Assets — Documentation Reference

## Overview

PNG icons of all 68 glyphs are generated from the canonical binary representation in `encoding/docs/glyph-table.tsv`. These are static images suitable for use in markdown documentation, diagrams, and printed materials.

## Location & Naming

```
docs/assets/glyphs/
├── 000.png  (empty glyph)
├── 001.png  (yī)
├── 003.png  (èr)
...
└── 511.png  (Full)
```

Filenames are zero-padded glyph IDs (e.g., `087.png` for glyph ID 87, the /tch/ phoneme).

## Generation

```bash
python3 direction/scripts/generate-glyph-pngs.py [size]
```

- **Default size:** 64px
- **Custom size:** `python3 direction/scripts/generate-glyph-pngs.py 128` for 128px
- **Output:** Overwrites existing PNGs in `docs/assets/glyphs/`

Requirements: ImageMagick (`convert` command)

## Usage in Documentation

### Markdown

Reference glyphs in markdown docs using relative paths:

```markdown
# Phoneme /tch/ (Glyph 87)

![glyph 87: /tch/](../docs/assets/glyphs/087.png)

This is the voiced affricate phoneme.
```

### From `language/`, `encoding/`, `display/` subdirectories

Adjust the path depth:

```markdown
![glyph 87](../../docs/assets/glyphs/087.png)
```

### In HTML

```html
<img src="../../docs/assets/glyphs/087.png" alt="glyph 87: /tch/" width="64">
```

```

```

## Integration with the Glyph Table

The interactive glyph table lives at:
- **File:** `docs/index.html`
- **Live:** https://marainkit.github.io/marain/
- **Generated by:** `direction/scripts/generate-glyph-table.py`

The PNG assets complement the table:
- **Interactive table** (`index.html`): Full details, font examples, status escalation colors
- **PNG icons** (markdown docs): Quick visual reference, embedded in discussion

## Design Philosophy

The PNG assets preserve the rotated 180° grid orientation (per Banks' vision — see `direction/glyph-grid-orientation.md`). Rendering is deterministic: same binary value always produces the same visual representation.

## Updating PNGs

If you modify the binary encoding of glyphs in the TSV, regenerate the PNGs:

```bash
python3 direction/scripts/generate-glyph-pngs.py
```

The hook in `.claude/settings.json` automatically regenerates `docs/index.html` when the TSV changes. You may want to also regenerate PNGs as part of the same workflow.
