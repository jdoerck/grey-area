Culture Theme — Style Guide

A contextual semantic design language. Token-driven, context-aware, legibility-first. This document captures the current state of the document / daylight / normal baseline and its proposed dark counterpart.

Baseline locked Dark mode proposed, not final v0.2

Overview

Design principles

Token-driven only

No hardcoded color, spacing, or type values anywhere in components. Everything traces back to a CSS custom property.

Legibility first

Designed for 60-second sustained reading, not first impression. Glyph disambiguation (Il1, O0, rn/m) required at every size.

States scale, don't shout

Warn and critical must be unambiguous, not visually loud. Escalation is contrast + structure, not color alone.

Context via CSS only

Mode switching uses data-mode attributes or class changes — no JavaScript style injection.

Stable typography

Typefaces do not change per context. Only weight, size, and spacing vary across modes.

Very Culture

Practical, dense, unpretentious. If tone and semantic weight are inside the glyph, you don't need theatrical staging.

Context model

All visual decisions derive from a three-axis context declaration. The current baseline is document / daylight / normal.

Color

Palette

All surfaces and text are warm-neutral — hue 30–42, low saturation. The palette reads as near-neutral but avoids the sterility of pure grey. Interactive and state colors are the only strong hues.

Typography

Type system

Typography is stable across all contexts — typefaces do not change per mode. Weight, size, and spacing vary; typeface selection does not.

Body & UI — Atkinson Hyperlegible

The quick brown fox jumps over a lazy dog. 0O Il1 rn/m

Designed by the Braille Institute for maximum legibility. Engineered disambiguation of ambiguous character pairs: I l 1, 0 O, rn m. Weights: 400, 700.

--font-body

Code & tokens — Intel One Mono

fn render(ctx: Context) → Glyph { 0O Il1 }

Designed for extended terminal and IDE use. Clean geometry, strong weight range, good disambiguation. Weights: 300 Light, 400 Regular, 500 Medium, 700 Bold.

Text treatments

Bold text for emphasis within prose. Italic for secondary or qualifying information. Strikethrough for removed content. Highlight for attention-without-alarm.

Linked text uses the accent color with a faded underline at rest, solid underline on hover.

Blockquote — for extended quotations or callout content. Left border is --line-strong, background is a tinted surface.

State escalation

Base-9 state scale

Status escalates across 9 levels (1–9), mapped to three bands. Colors below are approximate — the escalation system is designed but not finalized. The intent: escalation is contrast + structure, not color alone. A state-9 display in a critical context should be unmistakable; a state-1 display in normal context should read as almost invisible.

Code block specimen

/* token-driven display system */
:root {
  --surface-0:  hsl(42 24% 95%);   /* warm off-white */
  --text-0:     hsl(30 12% 18%);   /* dark warm brown */
  --accent:     hsl(210 42% 40%);  /* muted blue */
  --warn:       hsl(39 58% 48%);   /* amber */
  --critical:   hsl(5 63% 36%);    /* deep red */
}

function escalate(state: number): Token {
  if (state >= 8) return '--critical';
  if (state >= 5) return '--warn';
  return '--text-2';
}
