Lead paragraph — slightly larger, muted tone. Used for introductory copy or summaries that sit just below a heading.

Body text — this is the default paragraph style. It should be comfortable at reading length, with enough line height to keep the eye tracking across columns without effort or fatigue.

Small text — used for captions, footnotes, and secondary labels. Should remain legible at this reduced size without becoming brittle or crowded.

Caption — even smaller, for image notes, table captions, or fine print that is present but not primary.

Reading sample

The best font is usually the one that asks for the least attention.

It is easy to confuse novelty with quality. A font can feel crisp, strange, modern, or elegant in the first ten seconds and still become tiring when used across headings, controls, lists, labels, and paragraphs. That is why this page gives you enough text to read for a solid minute. A useful font should handle repetition well. It should not buckle under density, and it should not force personality onto ordinary interface tasks.

Try reading this without hurrying. Then move up to the menu and the action buttons. Then check the notices, the form fields, and the token line near the bottom. If one font feels better in the paragraph but worse in the controls, that matters. If another seems plain at first but turns out to be easier over time, that matters too. This is not about finding the most interesting typeface. It is about finding the one that wastes the least visual energy during normal use.

Pay attention to line shape, word rhythm, and the way the letters sit on the page. Some fonts feel tidy in isolation but become cramped when several interface elements compete for space. Others look awkward in a heading but become easier to trust in long reading. The right answer is rarely the most dramatic one.

What to watch for

  • Whether headings feel steady or overdesigned.
  • Whether body text stays calm across multiple paragraphs.
  • Whether small labels remain readable without becoming brittle.
  • Whether warning and approval notices read as signal, not theater.
  • Whether the overall page fades into the background after a minute.
A good reading font does not merely look good. It cooperates with the rest of the system.

JavaScript — context model

/**
 * context model — three axes define display state
 *
 * type:    'document' | 'hud'
 * viewing: 'daylight' | 'low-light'
 * status:  'normal' | 'warn' | 'critical'
 *
 * the combination selects a token set and a base-9 scale position.
 */

const CONTEXTS = {
  'document/daylight/normal':   { scale: 1, mode: 'light' },
  'document/daylight/warn':     { scale: 5, mode: 'light' },
  'document/daylight/critical': { scale: 8, mode: 'light' },
  'document/low-light/normal':  { scale: 1, mode: 'dark'  },
  'document/low-light/warn':    { scale: 5, mode: 'dark'  },
  'hud/low-light/normal':       { scale: 1, mode: 'hud'   },
  'hud/low-light/critical':     { scale: 8, mode: 'hud'   },
};

/**
 * applyContext — sets data-mode and data-scale on a target element.
 * in production this will live on document.documentElement.
 *
 * @param {HTMLElement} el
 * @param {'document'|'hud'} type
 * @param {'daylight'|'low-light'} viewing
 * @param {'normal'|'warn'|'critical'} status
 */
function applyContext(el, type, viewing, status) {
  const key = `${type}/${viewing}/${status}`;
  const ctx = CONTEXTS[key];

  if (!ctx) {
    console.warn(`[marain] unknown context: "${key}"`);
    return;
  }

  el.dataset.mode  = ctx.mode;
  el.dataset.scale = ctx.scale;
}

// usage
applyContext(document.documentElement, 'document', 'daylight', 'normal');

// state escalation — called when system status changes
function escalate(status) {
  const viewing = window.matchMedia('(prefers-color-scheme: dark)').matches
    ? 'low-light'
    : 'daylight';
  applyContext(document.documentElement, 'document', viewing, status);
}

JavaScript — encoding core (excerpt)

/**
 * core.js — shared encoding logic, no I/O
 * text → UTF-8 binary string → base-9 padded → 3×3 block SVG
 */

const BYTE_ROWS     = 9;
const BYTE_SIZE     = 3;   // pixels per bit cell
const BYTE_LENGTH   = 9;   // bits per word (base-9)
const WORD_CONSTANT = 9;

/**
 * textToBinary — convert a UTF-8 string to a binary string.
 * each character encodes to 8 bits; multi-byte chars expand accordingly.
 *
 * @param  {string} text
 * @returns {string}  binary string of '0' and '1'
 */
export function textToBinary(text) {
  const encoder = new TextEncoder();
  const bytes   = encoder.encode(text);
  return Array.from(bytes)
    .map(b => b.toString(2).padStart(8, '0'))
    .join('');
}

/**
 * buildSvg — full pipeline: text → SVG string.
 * pads binary to a multiple of 9, then renders each 9-bit window
 * as a 3×3 grid of  elements.
 *
 * @param  {string} text
 * @returns {string} SVG markup
 */
export function buildSvg(text) {
  let binary = textToBinary(text);

  // pad to multiple of BYTE_LENGTH
  const remainder = binary.length % BYTE_LENGTH;
  if (remainder !== 0) {
    binary = binary.padEnd(binary.length + (BYTE_LENGTH - remainder), '0');
  }

  const cols  = Math.ceil(binary.length / (BYTE_ROWS * BYTE_LENGTH));
  const width = cols  * BYTE_LENGTH * BYTE_SIZE;
  const height = BYTE_ROWS * BYTE_LENGTH * BYTE_SIZE;

  let rects = '';
  for (let i = 0; i < binary.length; i++) {
    const bit   = binary[i];
    const col   = Math.floor(i / (BYTE_ROWS * BYTE_LENGTH));
    const row   = i % (BYTE_ROWS * BYTE_LENGTH);
    const x     = (col * BYTE_LENGTH * BYTE_SIZE) + ((row % BYTE_SIZE) * BYTE_SIZE);  // simplified
    const y     = Math.floor(row / BYTE_SIZE) * BYTE_SIZE;
    const fill  = bit === '1' ? '#1a1612' : '#f5f2ed';
    rects += `<rect x="${x}" y="${y}" width="${BYTE_SIZE}" height="${BYTE_SIZE}" fill="${fill}"/>`;
  }

  return `<svg xmlns="http://www.w3.org/2000/svg" width="${width}" height="${height}">${rects}</svg>`;
}
